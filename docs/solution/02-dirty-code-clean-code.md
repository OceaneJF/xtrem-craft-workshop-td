# Clean code VS Dirty code

## Qu'est-ce que du code sale ?

- Un marécage ou on se perd
- Un code spaghetti
- Du code sans structure 
- Du code dupliqué
- Un patchwork mille fois reprisé
- Nommage flou
- Commentaires 
- Méthodes trop longues ( plus de 7 paramètres à une fonction )
- Historique flou (commit)
- Architecture / Design non structuré
- Responsabilités multiples
- Dépendances obsolètes
- 

## Qu'est-ce que du code propre ?

- Lisible => se lit comme une phrase, nommage métier, syntaxe standardisée
- Elégant => Assez simple et direct pour ne pas laisser de place au bug
- Facile à modifier / maintenable => Peut être modifié par un autre que son auteur
- Écrit avec soin => aucun problème évident ne saute aux yeux 
- Sans surprise => Le corps des méthodes est ce qu'on avait deviné
- Responsabilité unique (SRP)
- Dépendances isolées ( design pattern => adapteur , decorateur)
- Processus automatisés
- Pratiques définis (SOLID, KISS, CUPID, YAGNI, DRY ...)
